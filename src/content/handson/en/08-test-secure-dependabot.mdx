### 8.1 — Enable Dependabot

1. Repository **Settings** → **Security and quality** → **Advanced Security**
2. **Dependabot** section:
   - Enable **Dependency graph** (prerequisite for Dependabot)
   - Enable **Dependabot security updates**
   - Enable **Dependabot version updates** → an editor opens with a `.github/dependabot.yml` template
3. Change `package-ecosystem: ""` in the template to **`"npm"`** (this repo uses the pnpm / npm ecosystem). The final file should look like this:

```yaml
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
```

4. **Commit changes…** → choose **"Create a new branch and start a pull request"** → open the PR and merge it

<Aside type="negative">

**Heads up**: If `package-ecosystem` is left empty, the **Commit changes** button stays greyed out. Set it to a valid value like `"npm"` before committing. Also, the Ruleset you configured in Section 7.2 blocks direct commits to `main`, so the change has to go through a PR.

</Aside>

<Aside type="positive">

**Why it matters**: The two update types have different roles:

- **Security updates** — opens a PR the moment a vulnerability is added to the GitHub Advisory Database (reactive)
- **Version updates** — opens PRs to bump to the latest version on a schedule (e.g. weekly) defined in `.github/dependabot.yml`, regardless of vulnerabilities (proactive)

Enabling both covers vulnerability response *and* keeping dependencies up to date.

</Aside>
