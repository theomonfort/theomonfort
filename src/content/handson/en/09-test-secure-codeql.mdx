### 9.1 — Enable CodeQL

1. Repository **Settings** → **Security and quality** → **Advanced Security**
2. **Tools** → **CodeQL analysis** → **Set up** → **Default** → **Enable CodeQL**

<Aside type="negative">

**Heads up**: CodeQL only scans languages that exist on the default branch. If you enable CodeQL **after** merging your first Astro PR, automatic TypeScript / JavaScript scans will run on subsequent pushes and PRs.

</Aside>

<Aside type="positive">

**Why it matters**:

- **What it finds**: CodeQL uses static analysis to detect security issues like XSS and SQL injection.
- **When it runs**: A full scan of the default branch runs the moment you enable it, then on every push to the default branch, every PR, and on a weekly schedule (not just on PRs).
- **Where to see results**: All alerts show up in the repository's **Security** tab → **Code scanning**. On PRs, findings appear inline as comments on the affected lines.

</Aside>
