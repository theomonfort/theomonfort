### 6.1 — Switch the model to Claude Opus 4.8

Click the model selector at the bottom of the Copilot Chat panel and pick **Claude Opus 4.8**.

While you're there, also switch **"Default Approvals"** at the bottom of the panel to **"Bypass Approvals"**. Since we're working inside a Codespace sandbox, skipping the per-tool approval prompts is much faster.

<Aside type="positive">

**💡 Tip**: Opus-class models shine on complex design tasks and long-context conversations. For day-to-day editing and completion, other models are plenty, but for "scaffold the whole site in one shot" work like this section, Opus 4.8 gives more consistent quality.

</Aside>

### 6.2 — Switch to Plan mode

In Copilot Chat's mode switcher, select **"Plan"**.

<Aside type="positive">

**💡 Tip**: In Plan mode, Copilot focuses on **design and questions** rather than immediately editing files. It will push back on anything ambiguous in the requirements. Keep iterating with it until you're happy with the plan.

</Aside>

### 6.3 — Ask for an implementation plan

Copy the following prompt as-is and paste it into Copilot Chat:

````
Build a small Astro site (English only). Work on a new branch (e.g. feature/playbook-site).

- Create a grid-style index page that lists every entry in `src/content/playbook/*.md` as a tile (icon + title + short summary).
- Clicking a tile opens that entry directly in presentation mode (full-screen slide deck) — no intermediate detail page.
- Presentation-mode behavior:
  - Each `##` heading in the markdown becomes a new slide. The title slide is auto-generated from the frontmatter (`title`, `summary`, `icon`).
  - Each slide has its own unique URL (e.g. `/playbook/[slug]/[index]`). 1 URL = 1 slide — do not render every slide on the same page.
  - Keyboard: `←` / `→` to move between slides, `Esc` to return to the index.
  - A vertical navigation menu on the left of the screen. **This is NOT a slide list for the current entry — it lists the other playbook entries (the other `.md` files under `src/content/playbook/` such as GitHub, GitHub Copilot, Agent Skills, etc.) so the user can jump between entries.** Clicking an entry navigates to that entry's first slide while staying in presentation mode. Highlight the currently-open entry.
  - Use each entry's frontmatter `color` as the deck's accent color (slide border, headings).
- Render any ` ```mermaid ` code blocks inside the markdown as Mermaid diagrams (client-side is fine).

Once `npm run dev` confirms it works, commit and push your changes and open a pull request titled "Playbook site foundations".
````

Copilot may ask you a few questions (package manager, UI framework, deployment target, etc.). It should have read `copilot-instructions.md` and default to Astro / Tailwind / npm, but answer any gaps as they come up.

⏱ About 3 minutes

### 6.4 — Implement with Autopilot

Once you're happy with the plan, click **"Start with Autopilot"** to kick off the implementation.

That way, the moment Copilot opens the PR it's working on, the review fires automatically.

```bash
npm run dev
```

From the port-forwarding notification, click **"Open in browser"** and try it out.

<Aside type="positive">

**Try this**: Click through a few tiles on the grid and try out presentation mode. Move between slides with `←` / `→`, jump around by clicking the progress bar, and `Esc` returns to the index. Jot down anything that feels off, UI tweaks you'd like, or features you wish were there (we'll file them as Issues in the next step).

</Aside>

### 6.5 — File the improvements as Issues

Here's the key move — **instead of asking Copilot Chat to "fix this right now", file each improvement as a GitHub Issue**. In Step 11 (CODE: Cloud Agent) we'll have Cloud Agent implement some of them **in parallel, in one batch**, and in Step 13 (CODE: Copilot CLI) we'll knock out the remaining ones in parallel with `/fleet`.

Send the prompt below to Copilot Chat to file **8** Issues at once (the `github-issues` skill will structure them automatically):

<Aside type="reference">
The Issues below are **just one example** of the improvements you might want, based on how Copilot built the site. Depending on what Copilot actually shipped, some of them may not apply — skip those, and **feel free to swap in your own improvements** that make the site better the way you want.
</Aside>

```
Use the github-issues skill to create the following improvements as individual Issues in this repository.

1. Title: Add a left sidebar table of contents inside presentation mode
   Details: While presenting, pressing `T` should toggle a collapsible left sidebar listing every entry in the repository. The currently displayed entry is highlighted. Clicking another entry switches its presentation, staying in presentation mode and jumping to slide 1 of the new entry.

2. Title: Group the sidebar TOC by category
   Details: Build on Issue 1. Instead of a flat list, group entries by frontmatter `category` (introduction / plan / develop / review / secure / operate). Each group gets a pixel-font header with a category-specific color. Each entry shows as "number + icon + titleEn".

3. Title: Break the index page into category sections
   Details: Right now every entry sits in a single flat grid. Split them by `category`, with a heading and 1–2 lines of description per section (e.g. PLAN = set up the harness before you build, DEVELOP = write code with the AI, and so on).

4. Title: Add a "links" slide at the end of each entry
   Details: The frontmatter `links` (already grouped) should appear as the last slide of each entry, displayed grouped by their group, each with a subheading. Easy to reference while presenting.

5. Title: Replace the auto-generated title slide with an Octocat "in one line" hero slide
   Details: Replace the plain auto-generated title slide with a more visual hero. Put an Octocat (Mona) image on the left and a bordered dialog on the right showing frontmatter `summary` as the speech. Add a small pixel-font label "in one line" in the top-left of the slide.

6. Title: [Bug] Mermaid diagrams on non-initial slides aren't rendered
   Details: Putting a ` ```mermaid ` block on any slide past slide 1 leaves a blank area. Mermaid only runs once at mount; we need to re-run `mermaid.run()` against the active slide's container whenever the visible slide changes.

7. Title: [Bug] Slide content bunches up in the top-left, leaving empty space at bottom-right
   Details: Slide content currently anchors to the top-left, with a lot of whitespace at the bottom-right. Use flex/grid in the slide area to center horizontally and vertically — heading slightly above center, body near the middle.

8. Title: [Bug] Slide layout breaks and keyboard nav is unusable on mobile widths
   Details: Below `max-width: 768px`, keyboard navigation is unavailable and slides overflow the viewport. On narrow widths, disable presentation mode and fall back to a single vertical scroll view that renders the whole entry as a long-form page.
```

You can find the resulting Issues in the Issues tab of your repository. We'll have Cloud Agent / Copilot CLI implement them in the next sections.

⏱ About 3 minutes
