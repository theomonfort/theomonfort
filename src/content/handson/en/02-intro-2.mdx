This workshop uses the following GitHub repository:

**Project URL**: [https://github.com/theomonfort/Github-copilot-workshop](https://github.com/theomonfort/Github-copilot-workshop)

### Step 1: Create a repository from the template

1. Open the project URL in your browser
2. Click the **Use this template** button in the top right and select **Create a new repository**

![Use this template → Create a new repository](/theomonfort/handson/img/use-this-template-dropdown.png)

3. On the repository creation screen, configure the following:

#### Owner

- Select an **Organization where you can freely create test repositories**
- Requirements: an Organization with Copilot / Actions / GHAS / Codespaces enabled

#### Repository name

- Enter any name you like (e.g., `hands-on-yourname`)

#### Visibility

- Select **Private**

4. Click **Create repository**

![Click the Create repository button](/theomonfort/handson/img/create-repository-form.png)

### Step 2: Launch your dev environment with Codespaces

1. On your newly created repository's page, click the green **Code** button
2. Select the **Codespaces** tab
3. Click **Create codespace on main**

<Aside type="positive">

**⏳ Note**: Starting a Codespace can take a few minutes. Please wait for the DevContainer build to finish.

</Aside>

The DevContainer automatically sets up a Node.js 22 environment, with the following pre-installed:

- GitHub Copilot & Copilot Chat extensions
- GitHub CLI (`gh`)
- GitHub MCP Server (`.vscode/mcp.json`, auto-discovery enabled)
- Copilot CLI (`@github/copilot`)
- gh-aw (Agentic Workflow CLI)

### Step 3: Review the source content

Markdown files are placed in `src/content/playbook/`. These are the material for today's site.

In VS Code's explorer, open `src/content/playbook/` and take a peek at a few of the files. Later, you'll have Copilot turn them into a site that can be browsed as a "grid + detail + presentation mode" experience.
