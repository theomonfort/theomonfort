### 6.1 — モデルを Claude Opus 4.8 に切り替える

Copilot Chat 画面下部のモデルセレクターをクリックし、**Claude Opus 4.8** を選択します。

あわせて、画面下の **「Default Approvals」** を **「Bypass Approvals」** に切り替えてください。今回はサンドボックスである Codespace で作業しているので、ツール実行のたびに承認ダイアログを挟まずに進めた方が圧倒的に速いです。

<Aside type="positive">

**💡 ヒント**: 複雑な設計タスクや長文のコンテキストを扱う際は Opus 系モデルが強いです。日常の編集・補完は他のモデルで十分ですが、本セクションの「サイト全体を一気に組み立てる」ような作業では Opus 4.8 を使うと品質が安定します。

</Aside>

### 6.2 — Plan モードに切り替える

Copilot Chat のモード切替えから **「Plan」** を選択してください。

<Aside type="positive">

**💡 ヒント**: Plan モードでは Copilot は **設計と質問** に徹し、すぐにファイルを書き換えません。要件が曖昧な部分は逆質問してきます。納得できる計画になるまで対話を重ねましょう。

</Aside>

### 6.3 — 実装計画を依頼する

以下のプロンプトをそのままコピーして Copilot Chat に貼り付けてください：

````
小さな Astro サイトを作ってください（日本語のみ）。新しいブランチ（例：feature/playbook-site）で作業してください。

- `src/content/playbook/*.md` の全エントリを グリッド形式のインデックスページ にタイル（アイコン + タイトル + 短いサマリー）として一覧表示する。
- タイルをクリックすると、そのエントリが直接 プレゼンテーションモード（フルスクリーンのスライドデッキ）で開く ── 中間の詳細ページは無し。
- プレゼンテーションモードの挙動：
  - Markdown 内の各 `##` 見出しが 1 スライドになる。タイトルスライドは frontmatter（`title`、`summary`、`icon`）から自動生成。
  - 各スライドはそれぞれ固有の URL を持つ（例：`/playbook/[slug]/[index]`）。1 URL = 1 スライドで、全スライドを 1 ページにまとめてはいけない。
  - キーボード操作：`←` / `→` でスライド送り、`Esc` でインデックスに戻る。
  - 画面左側に縦型のナビゲーションメニューを表示する。**これは現在表示中のエントリー内のスライド一覧ではなく、他のプレイブックエントリー（`src/content/playbook/` 配下の他の `.md` ファイル ── GitHub、GitHub Copilot、Agent Skills など）への切替メニュー** とする。クリックするとそのエントリーのスライド 1 枚目に遷移し、プレゼンテーションモードは維持。現在開いているエントリーはハイライトする。
  - frontmatter の `color` をデッキのアクセントカラー（スライド枠線、見出し）として使う。
- Markdown 内の ` ```mermaid ` コードブロックは Mermaid 図 としてレンダリングする（クライアントサイドで可）。

`npm run dev` で動作確認できたら、変更をコミット & プッシュし、"Playbook site foundations" というタイトルでプルリクエストを作成してください。
````

Copilot がいくつか質問してくる場合があります（例：パッケージマネージャー、UI フレームワーク、デプロイ先など）。`copilot-instructions.md` を読んでいるはずなので Astro / Tailwind / npm を前提に答えてくれるはずですが、不足があれば回答してあげましょう。

⏱ 約3分

<Aside type="positive">

**ポイント**: 4.1 で書いた Stack ルールと、4.2 のフロントエンド design token がここで効きます。Plan が Astro + Tailwind + 指定カラーで提案されるはずです。

</Aside>

### 6.5 — 改善点を Issue 化する

ここで重要なポイント — **Copilot Chat に直接「ここを直して」と頼むのではなく、それぞれを GitHub Issue として登録します**。Step 11（CODE: Cloud Agent）で一部の Issue を Cloud Agent に **まとめて並列に** 実装してもらい、残りは Step 13（CODE: Copilot CLI）で `/fleet` を使って並列実装するためです。

Copilot Chat に以下のプロンプトを投げて、改善点を **8 件** Issue 化してください（`github-issues` skill が自動で構造化してくれます）：

<Aside type="reference">
以下の Issue は、Copilot がサイトを実装した結果に合わせてあなたが感じた改善点の **一例** です。実際の出来によっては当てはまらないものもあるので、合わないものは飛ばして、**自分が良くしたいと思った点を自由に Issue に置き換えてください**。
</Aside>

```
github-issues skill を使って、以下の改善点をこのリポジトリの Issue として個別に作成してください。

1. タイトル: プレゼンテーションモードに左サイドバーの目次を追加する
   詳細: プレゼン中に `T` キーで左側に折りたためる目次サイドバーを表示する。リポジトリ内の全エントリを一覧表示し、現在表示中のエントリをハイライトする。クリックで別エントリのプレゼンに切り替えられ、その際もプレゼンテーションモードを維持したまま新エントリの 1 枚目に遷移する。

2. タイトル: 目次サイドバーをカテゴリーで分類する
   詳細: 1 で追加した目次をフラット一覧ではなく frontmatter の `category`（introduction / plan / develop / review / secure / operate）ごとにグループ化する。各グループに pixel フォントの見出しを付け、カテゴリー別の色を使う。各エントリは「番号 + icon + titleEn」のフォーマットで表示する。

3. タイトル: インデックスページをカテゴリーセクションに分割する
   詳細: 現状は全エントリが 1 つのフラットなグリッドに並んでいる。`category` ごとにセクションを分け、各セクションに見出し + 1〜2 行の説明文を付ける（例：PLAN = 実装前のハーネスを整える、DEVELOP = AI と一緒にコードを書く など）。

4. タイトル: 各エントリの末尾に「リンク集」スライドを追加する
   詳細: frontmatter の `links`（group ごとにまとまっている）を、各エントリ最後のスライドとして group ごとに小見出しを付けて一覧表示する。プレゼン中に参考資料へ素早くアクセスできるようにする。

5. タイトル: 自動生成のタイトルスライドを Octocat「一言で」ヒーロースライドに置き換える
   詳細: 現状の素朴な自動生成タイトルスライドを、よりビジュアルなヒーロースライドに置き換える。左に Octocat（Mona）キャラクター画像、右に枠付きのダイアログを配置し、frontmatter の `summary` をセリフとして表示する。スライド左上には pixel フォントで小さく「一言で」のラベルを置く。

6. タイトル: 【バグ】2 枚目以降のスライドの Mermaid 図が描画されない
   詳細: 1 枚目以外のスライドに ` ```mermaid ` ブロックを置くと空白のまま描画されない。Mermaid は初回マウント時しか実行されないため、スライド切り替え時に該当領域に対して `mermaid.run()` を再実行する必要がある。スライドが表示状態になった瞬間に再レンダリングするように修正する。

7. タイトル: 【バグ】スライド本文が左上に偏って右下に大きな空白ができる
   詳細: 現状コンテンツが左上に寄り、右下に大きな空白ができてしまう。スライド領域内で flex/grid を使い、見出しと本文を縦横ともに中央へ揃える。タイトルはやや上、本文は中央付近に配置する。

8. タイトル: 【バグ】モバイル幅でスライドが崩れキーボード操作も使えない
   詳細: `max-width: 768px` 以下ではキーボードナビゲーションが使えず、スライド本体も画面からはみ出してしまう。狭い画面ではプレゼンテーションモードを無効化し、エントリ全体を 1 枚の縦長ページとしてスクロール表示できるフォールバックを追加する。
```

作成された Issue のリストはリポジトリの Issues タブで確認できます。次の章以降で Cloud Agent / Copilot CLI に実装させます。

⏱ 約3分
