### 6.1 — モデルを Claude Opus 4.6 に切り替える

Copilot Chat 画面下部のモデルセレクターをクリックし、**Claude Opus 4.6** を選択します。

<Aside type="positive">

**💡 ヒント**: 複雑な設計タスクや長文のコンテキストを扱う際は Opus 系モデルが強いです。日常の編集・補完は他のモデルで十分ですが、本セクションの「サイト全体を一気に組み立てる」ような作業では Opus 4.6 を使うと品質が安定します。

</Aside>

### 6.2 — Plan モードに切り替える

Copilot Chat のモード切替えから **「Plan」** を選択してください。

<Aside type="positive">

**💡 ヒント**: Plan モードでは Copilot は **設計と質問** に徹し、すぐにファイルを書き換えません。要件が曖昧な部分は逆質問してきます。納得できる計画になるまで対話を重ねましょう。

</Aside>

### 6.3 — 実装計画を依頼する

以下のプロンプトをそのままコピーして Copilot Chat に貼り付けてください：

````
小さな Astro サイトを作ってください（日本語のみ）。新しいブランチ（例: feature/playbook-site）で作業してください。

- `src/content/playbook/*.md` の各エントリを 1 枚のタイル（アイコン + タイトル + 短いサマリー）で並べた **グリッド型のインデックスページ** を作成。
- タイルをクリックすると、そのエントリの詳細ページ（マークダウン全文・スクロール可）に遷移。
- 詳細ページで `P` キーを押すと **プレゼンテーションモード** をトグル：
  - `##` 見出しごとに 1 スライドに分割（分割規約は `md-slides-creator` Skill を参照）。
  - `←` `→` で同一エントリ内のスライドを移動。
  - `Esc` で通常表示に戻る。
- プレゼンテーションモードでは左サイドバーに全エントリの目次を表示し、現在のエントリをハイライト。

最後に、`pnpm dev` で動作確認できたら変更をコミット・プッシュし、**「Playbook サイトの基盤」** というタイトルで Pull Request を作成してください。
````

Copilot がいくつか質問してくる場合があります（例：パッケージマネージャー、UI フレームワーク、デプロイ先など）。`copilot-instructions.md` を読んでいるはずなので Astro / Tailwind / pnpm を前提に答えてくれるはずですが、不足があれば回答してあげましょう。

### 6.4 — Autopilot で実装する

Plan に納得したら **「Start with Autopilot」** をクリックして実装を開始します。

![Start with Autopilot](/theomonfort/handson/img/start-with-autopilot.png)

⏱ 約15〜20分

Copilot が以下を自動で実行します：

- Astro プロジェクトの初期化（pnpm）
- `src/content.config.ts` で playbook collection を定義
- Presentation UI（スライド表示）
- キーボード操作（← → ↑ ↓）
- 左サイドバー（セクション目次 + 現在セクションのハイライト）
- frontmatter の `color` に基づくセクションごとのテーマカラー
- Mermaid 図のレンダリング
- design token に沿った Tailwind スタイル

実装が完了したら開発サーバーを起動して確認しましょう：

```bash
pnpm dev
```

ポートフォワーディングの通知から **「ブラウザで開く」** をクリックして動作確認してください。

<Aside type="positive">

**やってみよう**: スライドを ← → ↑ ↓ で操作してみてください。挙動がおかしい箇所、UI の改善点、追加したい機能などをメモしておきましょう。

### 6.5 — 改善点を Issue 化する

ここで重要なポイント — **Copilot Chat に直接「ここを直して」と頼むのではなく、それぞれを GitHub Issue として登録します**。Part 12（CODE: CLI）で、これらの Issue を Copilot CLI に **まとめて並列に** 実装してもらうためです。

Copilot Chat に以下のプロンプトを投げて、改善点を **6 件** Issue 化してください（`github-issues` skill が自動で構造化してくれます）：

```
github-issues skill を使って、以下の改善点をこのリポジトリの Issue として個別に作成してください。

1. タイトル: スライドモードの左サイドバーをカテゴリー付きの目次にする
   詳細: 現在のサイドバーはエントリをフラットに並べているだけ。frontmatter の `category`（introduction / plan / develop / review / secure / operate）ごとにグループ化し、各グループに pixel フォントのヘッダー（色は category 別）を付ける。各エントリは「番号 + icon + titleEn」のフォーマットで、現在表示中のエントリをハイライト。

2. タイトル: グリッドのインデックスページをカテゴリーごとのセクションに分ける
   詳細: 現在は全エントリを 1 つのフラットなグリッドで表示している。`category` ごとにセクション分割し、各セクションに見出し + 1〜2 行の説明文を表示する（例: PLAN = 実装前の harness を整える、DEVELOP = AI と一緒にコードを書く など）。

3. タイトル: 各エントリの最後に「リンク集」スライドを追加
   詳細: frontmatter の `links`（group ごとにまとまっている）を、各エントリ最後のスライドとしてグルーピング表示する。group ごとに小見出しを付け、プレゼン中に参照しやすくする。

4. タイトル: スライド内テキストを縦横ともに中央揃えにする
   詳細: 現状コンテンツが左上に寄って空白が右下に偏る。スライド領域内で flex/grid を使い、見出しと本文を視覚的に中央へ。タイトルはやや上、本文は中央付近に揃える。

5. タイトル: スライド上部に進捗バーを追加
   詳細: 画面最上部に細い neon-cyan の進捗バーを表示し、全体のどこにいるかを示す。セクション境界に区切りを入れ、現在セクション内の進捗も分かるようにする。

6. タイトル: 各エントリの 1 枚目に Octocat の「一言で」サマリースライドを追加
   詳細: 各エントリ冒頭のスライドとして、画面左に Octocat（Mona）キャラクター画像 + 画面右に枠付きのダイアログを配置し、frontmatter の `summary` を要約として表示する。スライド左上には pixel フォントで小さく「一言で」のラベルを置く。
```

作成された Issue のリストは [リポジトリの Issues タブ](#) で確認できます。次の章以降で Cloud Agent / Copilot CLI に実装させます。
