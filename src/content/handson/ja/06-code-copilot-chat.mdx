### 6.1 — モデルを Claude Opus 4.8 に切り替える

Copilot Chat 画面下部のモデルセレクターをクリックし、**Claude Opus 4.8** を選択します。

あわせて、画面下の **「Default Approvals」** を **「Bypass Approvals」** に切り替えてください。今回はサンドボックスである Codespace で作業しているので、ツール実行のたびに承認ダイアログを挟まずに進めた方が圧倒的に速いです。

<Aside type="positive">

**💡 ヒント**: 複雑な設計タスクや長文のコンテキストを扱う際は Opus 系モデルが強いです。日常の編集・補完は他のモデルで十分ですが、本セクションの「サイト全体を一気に組み立てる」ような作業では Opus 4.8 を使うと品質が安定します。

</Aside>

### 6.2 — Plan モードに切り替える

Copilot Chat のモード切替えから **「Plan」** を選択してください。

<Aside type="positive">

**💡 ヒント**: Plan モードでは Copilot は **設計と質問** に徹し、すぐにファイルを書き換えません。要件が曖昧な部分は逆質問してきます。納得できる計画になるまで対話を重ねましょう。

</Aside>

### 6.3 — 実装計画を依頼する

以下のプロンプトをそのままコピーして Copilot Chat に貼り付けてください：

````
Build the Playbook landing page in Japanese only (work on a new branch: `feature/playbook-landing-page`).

Requirements:
* Use `src/content/playbook/ja/*.md` as the source.
* Create `/playbook` with cards grouped by `category` (2 columns of cards).
* Use `src/lib/playbook-categories.ts` for category labels, actor names, avatars, descriptions, colors etc.
* Each card shows icon, title, summary, and uses the `accent` frontmatter for colors.
* Add decorative floating assets near the matching SDLC category (about 450px wide):
    * introduction → `public/planet-introduction.webp`
    * plan → `public/planet-plan.webp`
    * develop → `public/planet-develop.webp`
    * review → `public/planet-review.webp`
    * secure → `public/planet-secure.webp`
    * operate → `public/planet-operate.webp`
    * administration → `public/planet-administration.webp`
* Clicking a card opens a detail page that renders the full Markdown.
* Detail page has a “Back to Playbook” button.

Add a top “SDLC navigation chain” above the introduction section:
  Plan → Develop → Review → Test & Secure → Operate
  Each item should be a bordered pixel button using the matching category color, with arrows between items, and should link to the category anchor.

Once `npm run dev` confirms it works, commit and push your changes and open a pull request titled "Playbook site foundations".
````

Copilot がいくつか質問してくる場合があります（例：パッケージマネージャー、UI フレームワーク、デプロイ先など）。`copilot-instructions.md` を読んでいるはずなので Astro / Tailwind / pnpm を前提に答えてくれるはずですが、不足があれば回答してあげましょう。

⏱ 約3分

<Aside type="positive">

**ポイント**: 4.1 で書いた Stack ルールと、4.2 のフロントエンド design token がここで効きます。Plan が Astro + Tailwind + 指定カラーで提案されるはずです。

</Aside>

### 6.4 — Autopilot で実装する

Plan に納得したら **「Start with Autopilot」** をクリックして実装を開始します。

![Start with Autopilot](/theomonfort/handson/img/start-with-autopilot.png)

⏱ 約15分

<Aside type="positive">

**💡 並行して進めよう**: Copilot が実装している間は手が空くので、先に [Step 7（REVIEW: Copilot Code Review）](../review-copilot-code-review/) に進んで **Automatic Copilot Code Review** を有効化しておきましょう。設定しておけば、Copilot が今作成中の PR がオープンされた瞬間に自動でレビューが走ります。

</Aside>

実装が完了したら開発サーバーを起動して確認しましょう：

```bash
npm run dev
```

ポートフォワーディングの通知から **「ブラウザで開く」** をクリックして動作確認してください。

<Aside type="positive">

**やってみよう**: `/playbook` を開き、カテゴリごとの色、デスクトップでカードが 2 カラムになっていること、planet assets が十分大きく見えることを確認します。いくつかタイルをクリックし、詳細ページで Markdown 全文が表示されること、Back to Playbook ボタンで戻れることも確認しましょう。挙動がおかしい箇所、UI の改善点、追加したい機能などをメモしておきましょう（次のステップでまとめて Issue 化します）。

</Aside>

### 6.5 — 改善点を Issue 化する

ここで重要なポイント — **Copilot Chat に直接「ここを直して」と頼むのではなく、それぞれを GitHub Issue として登録します**。Step 11（CODE: Cloud Agent）で一部の Issue を Cloud Agent に **まとめて並列に** 実装してもらい、残りは Step 13（CODE: Copilot CLI）で `/fleet` を使って並列実装するためです。

Copilot Chat に以下のプロンプトを投げて、改善点を **8 件** Issue 化してください（`github-issues` skill が自動で構造化してくれます）：

<Aside type="reference">
以下の Issue は、Copilot がサイトを実装した結果に合わせてあなたが感じた改善点の **一例** です。実際の出来によっては当てはまらないものもあるので、合わないものは飛ばして、**自分が良くしたいと思った点を自由に Issue に置き換えてください**。
</Aside>

```
github-issues skill を使って、以下の改善点をこのリポジトリの Issue として個別に作成してください。

1. タイトル: Playbook landing page に category description と actor を追加する
   詳細: `src/lib/playbook-categories.ts` を使い、各カテゴリの actor avatar、actor name、description をカード一覧の上に表示する。

2. タイトル: planet の配置とサイズを調整する
   詳細: 各 `planet-*.webp` を大きな装飾背景として見せる。テキストを隠したりクリックを邪魔したりしないように、desktop / mobile の両方で調整する。

3. タイトル: SDLC navigation chain に active styling を追加する
   詳細: 上部の Plan → Develop → Review → Test & Secure → Operate ボタンにカテゴリカラーの hover / focus state を付ける。可能なら現在表示中のカテゴリをハイライトする。

4. タイトル: Playbook detail page の typography を改善する
   詳細: 詳細ページで見出し、テーブル、リスト、コードブロック、blockquote、リンクが landing page と同じレトロテーマで読みやすく表示されるようにする。

5. タイトル: 詳細ページで Mermaid 図をレンダリングする
   詳細: Markdown 詳細ページに ` ```mermaid ` ブロックがある場合、通常のコードブロックではなく Mermaid 図として表示する。

6. タイトル: 【バグ】`/theomonfort/` base path で public asset が読み込めない
   詳細: icons、planets、images が `/` 直下だけでなく `/theomonfort/` 配下で配信された場合も正しく読み込めるようにする。

7. タイトル: 【バグ】モバイルで planet が重なる / card が窮屈になる
   詳細: 狭い画面ではカードを 1 カラムにし、planet を縮小または背景側に逃がして、本文の読みやすさを保つ。

8. タイトル: Playbook card の簡易検索 / フィルターを追加する
   詳細: タイトルと summary でカードを絞り込める client-side search box を追加する。category grouping は分かりやすく維持する。
```

作成された Issue のリストはリポジトリの Issues タブで確認できます。次の章以降で Cloud Agent / Copilot CLI に実装させます。

⏱ 約3分
