**Macro Trends and Emerging (Relevant) Tech**
_There is a robust ecosystem around GitHub that impacts our positioning and traction. This section brings select developments and pertinent analysis of technologies in our sphere._ 

* **TSMC**, a chip foundry, [announced Q4FY25](https://investor.tsmc.com/english/encrypt/files/encrypt_file/reports/2026-01/3e49621566a3ca53bdf8aee2586929b666c17fd6/4Q25EarningsRelease.pdf) earnings of $33.7Bn (+25%YoY) driven by continued strong demand for AI related products.

* **Minimax**, a Chinese AI Lab, who recently released MiniMax 2.1 (see Mona Monitor [43](https://github.com/github/Business-Insights/discussions/52), [39](https://github.com/github/Business-Insights/discussions/48)) raised $619M in [an over-subscribed IPO](https://ir-upload.realxen.net/iis/0100/uploads/iis/2026/11987013-0.PDF), while [**Z.A**I](http://z.ai/), makers of the QLM model (see Mona Monitor [39](https://github.com/github/Business-Insights/discussions/48)) raised $558M in its [IPO](https://www.prnewswire.com/news-releases/chinas-agi-pioneer-and-leader-zai-listed-on-hong-kong-stock-exchange-302656265.html). Both companies floated on the Hong Kong exchange.  

* Alison Wagonfeld [joined **Nvidia** as CMO](https://www.linkedin.com/posts/alisonwagonfeld_sharing-some-personal-news-after-nearly-activity-7415073758190944256-SUVX). Wagonfeld was previously VP of Marketing at **Google** Cloud.

**Competitor News**
_This section covers select updates and analysis from known and emerging competitors._  

**AI**

* **Anthropic** had a very busy few weeks
  * Anthropic [launched Anthropic Labs](https://www.anthropic.com/news/introducing-anthropic-labs), which will incubate experimental ideas. CPO Mike Krieger is moving over to run labs, while Ami Vora, who recently joined as product lead, steps into the CPO role. Vora was a VP of Product at **Facebook** in the past. 
  * **Anthropic** [limited access to Claude Code](https://x.com/trq212/status/2009689809875591565) via tools like **OpenCode**. Various people got annoyed, but Claude Code usage remains strong. However, this move does, like previous moves around pricing with Cursor (see Mona Monitor [33](https://github.com/github/Business-Insights/discussions/42)) and disabling model access with Windsurf (see Mona Monitor [28](https://github.com/github/Business-Insights/discussions/36)) provide talking points to support our open platform and model choice approach. 
  * **Anthropic** [announced Claude Cowork](https://claude.com/blog/cowork-research-preview), tooling for using Claude Code in a more general manner. It is currently a research preview and only available to users on their max plan. 
  * Claude Code [announced a tool search tool](https://platform.claude.com/docs/en/agents-and-tools/tool-use/tool-search-tool) which introduces lazy loading of tools with MCP to help reduce context required when using multiple MCP servers.   
  * The Claude Code extension for VS Code [moved to GA](https://x.com/claudeai/status/2013704053226717347?s=20).
  * **Anthropic's** enterprise and executive PR efforts stepped up with [a piece on Claude Code](https://www.wsj.com/tech/ai/anthropic-claude-code-ai-7a46460e) in the **Wall Street Journal**, and [sponsoring](https://www.economist.com/insider) The Insiders feature of **The Economist**. 

* **OpenAI** [released Codex 5.2](https://openai.com/index/introducing-gpt-5-2-codex/). As is customary, various vendors immediately announced support, including GitHub, [**Windsurf**](https://windsurf.com/blog/gpt-5-2-codex), [**Cline**](https://cline.bot/blog/cline-3-51-0-gpt-5-2-codex-support). **Cursor** once again relegated the announcement to [a post on X](https://x.com/cursor_ai/status/2011500027945033904?s=20), but [published a post on long running agents](https://cursor.com/blog/scaling-agents) which flagged their use of Codex 5.2. 

* **Cursor** [added new features to their CLI](https://cursor.com/changelog/cli-jan-16-2026) including one click MCP authentication, a cloud hand off mode (similar to [/delegate](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/use-copilot-cli#delegate-tasks-to-copilot-coding-agent) in the Coplot CLI) and plan mode. 

* Skills arrived as a table stakes feature, with [**Cline**](https://cline.bot/blog/cline-3-48-0-skills-and-websearch-make-cline-smarter), [**Windsurf**](https://docs.windsurf.com/windsurf/cascade/skills) and [**Kiro**](https://kiro.dev/docs/cli/custom-agents/configuration-reference/#skill-resources) among the vendors announcing support. an AI coding tool, [announced skills functionality](https://cline.bot/blog/cline-3-48-0-skills-and-websearch-make-cline-smarter).  

* **Vercel** launched [skills.sh](http://skills.sh/), a curated registry of skills which can be installed with multiple agents. More details in the [launch blog](https://vercel.com/changelog/introducing-skills-the-open-agent-skills-ecosystem). 

* **Amp**, the ad supported AI coding tool (see Mona Monitor [38](https://github.com/github/Business-Insights/discussions/47)), has [removed their version of tab complete](https://ampcode.com/news/tab-tab-dead), pointing users at **Cursor**, **Copilot** or **Zed** if they need tab. This is part of their positioning to be at the forefront of agents. Amp also [removed TODOs](https://ampcode.com/news/todos-are-done) as a construct. It is unclear if they asked humans, who gravitate towards todo lists for all demos, about this.  

* **Kilo Code**, an AI Coding Assistant founded by ex GitLab CEO Sid Sijbrandij, [announced a slack integration](https://blog.kilo.ai/p/announcing-kilo-for-slack). 

* **Mintlify**, an AI documentation tool, [proposed install.md](https://www.mintlify.com/blog/install-md-standard-for-llm-executable-installation) as a standard for how llms show install agents. It is unclear where, if anywhere, this will go. However, Mintlify has been successful pushing llms.txt as a standard. 

* **Cubic**, an AI code review vendor, [had a launch week](https://www.cubic.dev/blog/cubic-launch-week-recap), including a major update to their core product with [Cubic 2.0](https://www.cubic.dev/blog/cubic-2.0), [a new CLI](https://www.cubic.dev/blog/new-cubic-cli) and [incremental PR reviews](https://www.cubic.dev/blog/prs-keep-changing.-now-cubic-keeps-up). 

* **EntelligenceAI**, an AI code review and metrics vendor, [announced a $5M round](https://entelligence.ai/blogs/entelligence-ai-raises-5m-to-reimagine-how-engineering-teams-ship) led by Mayfield. 

* **Linear**, a project and product planning vendor, [announced an AI Code Review tool](https://x.com/linear/status/2013643099147248054). Access is currently restricted, with the option to sign up for a waitlist. 

* **Sentry**, a developer focused observability vendor, is running an AI Code Review beta program for their [new Seer AI offering](https://docs.sentry.io/product/ai-in-sentry/seer/). 

* **Replit**, a vibe coding vendor, [announced a new Pro tier ](https://blog.replit.com/pro-plan)at $100/pupm, which will replace the existing $35/pupm tier. The vibes are more expensive. 

* **Emergent**, a vibe coding vendor, [announced a $70M Series B](https://finance.yahoo.com/news/emergent-raises-70m-khosla-ventures-180000776.html) led by Softbank and Khosla ventures. Emergent immediately revealed part of how they plan to increase their burndown rate by cutting the price of their standard tier from $20pm to $5pm. The early stage funding vibes continue to be good. 

* **Lovable**, a vibe coding vendor, [introduced a new credit top up plan](https://docs.lovable.dev/introduction/plans-and-credits#credit-top-ups) for users on their pro and business tier. Credits are valid for 12 months.


**DevOps**

* **GitLab** [announced the general availability of GitLab Duo Platform](https://about.gitlab.com/blog/gitlab-duo-agent-platform-is-generally-available/) and [a new credit based pricing model](https://about.gitlab.com/blog/introducing-gitlab-credits/) for their AI offerings. 

* **GitLab** [announced a new CTO](https://www.linkedin.com/posts/gitlab-com_meet-our-chief-technology-officer-siva-padisetty-activity-7416480637404008448-uJX9), Siva Padisetty, who replaces Sabrina Farmer.  Previously worked with current GitLab CEO Bill Staples at **New Relic**. 

* **GitLab** is [introducing mandatory MFA for users](https://about.gitlab.com/blog/strengthening-gitlab-com-security-mandatory-multi-factor-authentication/). Users who use a social login (such as **GitHub**) or their employer's SSO solution are exempt. 

* **Docker** [quietly released a new experimental sandboxing feature for AI Agents](https://docs.docker.com/ai/sandboxes/), with Claude as the first example. This should be viewed in context with Claude's own sandboxing on Mac released as part of Claude Cowork, using Apple's virtualization framework. Simon Willison [has a great write up on this](https://simonwillison.net/2026/Jan/12/claude-cowork/).

**Security**

* **Checkmarx**, a security vendor, [announced their AI Query Builder for SAST](https://checkmarx.com/blog/ai-query-builder-for-sast-now-generally-available/) is now generally available. 

* **Aikido**, a security vendor, [announced a $60M Series B](https://www.aikido.dev/blog/aikido-funding-series-b), led by DST Global. Aikido competes directly against GHAS with their Aikido Code offering. 

* **DepthFirst**, positioned as an AI Native security vendor, [announced a $40M Series A](https://depthfirst.com/post/depthfirst-announces-40m-series-a-to-secure-the-worlds-software) led by Accel. Their platform integrates directly with GitHub. There is no documentation or demos for the offering available at this time. 

* **Mend** [published an AI Security Maturity Survey and associated Compliance Checklist](https://www.mend.io/blog/introducing-mend-io-ai-security-maturity-survey/) for organizations to evaluate their current status and provide a path forward. Readers will not be surprised to learn that Mend’s offerings support that path forward. That said, most enterprises love a maturity framework as it gives them a line in the sand to start from. 

* **GitGuardian** [published a classic big numbers marketing post](https://blog.gitguardian.com/gitguardian-closes-2025-with-strong-enterprise-momentum-protecting-millions-of-developers-worldwide/), highlighting that 115K enterprise developers are using their platform across 610K enterprise numbers. GitHub is working on a similar report.  

* **JetBrains** [announced capabilities to set global configuration rules for Qodan](https://blog.jetbrains.com/qodana/2026/01/introducing-global-project-configuration-one-place-to-manage-all-your-qodana-rules/)a, their SCA offering ([docs](https://www.jetbrains.com/help/qodana/2025.3/global-configuration.html)).

**Data Points and Other**

* **SonaType** released their [latest OpenSource Malware Index,](https://www.sonatype.com/blog/open-source-malware-index-q4-2025-automation-overwhelms-ecosystems) and focused heavily on framework and dependency abuse in NPM with a stat that “99.8% of all Q4 malware originated from npm”

* **Steve Yegge** [released Gas Town](https://steve-yegge.medium.com/welcome-to-gas-town-4f25ee16dd04), an orchestration framework, which he described as kubernetes for agents. This prompted analyst firm **RedMonk** to declare that [swarm coding is going to be the default approach](https://redmonk.com/sogrady/2026/01/08/tide-of-agents/).  

* **Google** [published a stat](https://cloud.google.com/blog/topics/public-sector/new-google-public-sector-research-shows-that-nearly-90-of-federal-agencies-are-already-using-ai/), citing research they had commissioned with Government Executive, that 90% of federal agencies in the US are using AI. The actual survey is not available. 

* **Cloudflare** [acquired Astro Technology Company](https://blog.cloudflare.com/astro-joins-cloudflare/), the company behind the popular Astro web framework, and [Human Native](https://blog.cloudflare.com/human-native-joins-cloudflare/), a company specializing in transforming multimedia into useful data for AI. Human Native fits into Cloudflare's broader [pay per crawl](https://blog.cloudflare.com/introducing-pay-per-crawl/) vision.
