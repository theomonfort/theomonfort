# GitHub Copilot CLI

📖 **公式ドキュメント**: [Copilot CLIのインストール](https://docs.github.com/en/copilot/how-tos/copilot-cli/set-up-copilot-cli/install-copilot-cli)

## クイックスタート

```bash
# インストール (macOS/Linux)
brew install copilot-cli

# プロジェクトディレクトリで起動
cd your-project && copilot

# 初回: 認証
/login
```

## Copilot CLIとは

Copilot CLIは、**ターミナル上で動作する自律型AIエージェント**です。計画、実行、結果の確認、修正の繰り返し — すべてをコマンドラインから行います。

- **ネイティブMCPサーバー** — GitHub MCPサーバーが組み込まれており、RESTおよびGraphQL APIに直接アクセス可能。Issue照会、PRのdiff確認、CIログの確認、リポジトリ横断コード検索 — すべて自然言語で実行できます。
- **フルシェルアクセス** — パッケージのインストール、スクリプトの実行、複雑なワークフローの連携、エラーの自動修正が可能。APIアクセスと組み合わせることで、データの照会から実行までのエンドツーエンドの自動化を実現します。
- **Fleetモード** — 独立したタスクを並行処理する複数のサブエージェントを起動。マルチファイル操作が大幅に高速化されます。
- **メモリとコンテキスト制御** — セッション間で情報を保持し、コンテキストウィンドウの使用状況を監視・管理できます。AIの作業メモリに対する可視性と制御を提供します。
- **ターミナルがあればどこでも動作** — SSH、リモートサーバー、コンテナ、ヘッドレスマシン。IDEは不要。軽量で即座に起動、完全キーボード操作。
- **`~/.copilot/`でカスタマイズ可能** — 個人用のスキル、エージェント、指示を定義すれば、マシン上のすべてのプロジェクトで利用可能。どこでも使える自分だけのツールボックス。

## セットアップ

### ユーザーレベル設定 (`~/.copilot/`)

**個人用**の設定 — マシン上のすべてのプロジェクトに適用されます：

```
~/.copilot/
├── copilot-instructions.md    # すべてのプロジェクトに適用されるグローバルな指示
├── agents/                     # 個人用エージェント、どこでも利用可能
│   └── my-agent.md
└── skills/                     # 個人用スキル、どこでも利用可能
    └── my-skill/
        └── SKILL.md
```

### リポジトリレベル設定 (`.github/`)

**プロジェクト固有**の設定 — リポジトリに含まれます：

```
.github/
├── copilot-instructions.md    # プロジェクト構造、規約、設定
├── agents/                     # リポジトリ固有のエージェント
│   └── my-agent.md
└── skills/                     # リポジトリ固有のスキル
    └── my-skill/
        └── SKILL.md
```

CLIは両方のレベルをマージします — 個人用ツールとリポジトリが提供するツールの両方が使えます。

> 💡 **Theo's Tip**: `~/.copilot/`フォルダはデフォルトで非表示です。macOS Finderでは**`Cmd+Shift+.`**で表示を切り替えられます。ターミナルでは`ls -a`を使ってください。

> 💡 **Theo's Tip**: CLIはエージェントとスキルを2つの場所から読み込みますが、**YAMLフロントマター**（`---`区切り）が必須です。`~/.copilot/skills/`のスキルはどのプロジェクトでも利用可能。`.github/skills/`のスキルはそのリポジトリ内でのみ利用可能。エージェントも同様です。`/agent`や`/skills`に表示されない場合は、フロントマターを確認してください。

## Fleetモード

`/fleet`は**複数のサブエージェントを並行起動**し、独立したタスクを同時処理します。この機能は**CLI限定**です — VS Code Chatでは並行サブエージェントはサポートされていません。1つのエージェントが順番に処理する代わりに、Fleetモードは作業を分散させ、複数ファイル・フォルダにまたがる操作を大幅に高速化します。

### 使用例

```
/fleet 各トップレベルフォルダにSUMMARY.mdを作成して。中身はファイル数、行数、サブフォルダ一覧、内容の要約
```

このプロンプト1つで、トップレベルフォルダごとに1つのサブエージェントが起動。各エージェントが独立してファイル数、行数をカウントし、サブフォルダを一覧化し、内容を要約します — すべて同時に実行されます。

### Fleetの進捗確認

`/task`を使って、各サブエージェントの状態をリアルタイムで確認できます：実行中、完了済み、生成された成果物を確認できます。

> ⚠️ **Theo's Tip**: `/task`ウィンドウを長時間表示したままにしないでください。サブエージェントが承認を求める場合があり（例：ファイル変更の承認）、`/task`を見続けていると承認プロンプトを見逃し、Fleetが入力待ちで停止してしまいます。

## メモリとセッション管理

### コンテキストウィンドウの制御

CLIはコンテキストウィンドウの**完全な可視性と制御**を提供します — VS Code Chatにはない機能です：

| コマンド | 機能 |
|---------|------|
| `/context` | コンテキストウィンドウの使用量を正確に表示（%とトークン数） |
| `/compact` | 会話を要約してコンテキスト領域を回復 |

コンテキスト使用率が高くなると、エージェントは以前の詳細を見失うことがあります。応答の質が低下したと感じたら：
1. CLIに**重要な情報をメモリに保存**するよう依頼（`store_memory`でセッション間の知識を永続化）
2. **新しいセッションを開始** — 保存されたメモリが自動的に読み込まれます

> 💡 **Theo's Tip**: セッション開始直後に`/compact`を実行しても、あまり空き容量は増えません。システムプロンプト、ツール定義、読み込まれた指示は固定量のコンテキストを消費するため、このベースラインは圧縮不可能です。

### セッション管理

| コマンド | 機能 |
|---------|------|
| `/session` | 現在のセッションのチェックポイントと履歴を表示 |
| `/resume` | 以前のセッションに切り替えまたは再開 |
| `/rename` | 現在のセッション名を変更して後で見つけやすくする |

セッションは作業中に自動的にチェックポイントが作成されます。`/resume`で以前のセッションに戻れます — タスクの切り替えや中断した作業の再開に便利です。

## その他のヒントとテクニック

### `/help` — チートシート

いつでも`/help`を実行して、利用可能なすべてのコマンドとキーボードショートカットを確認できます。CLIでできることを最も早く知る方法です。

### `/share` — セッションのエクスポート

現在の会話を**GitHub Gist**またはmarkdownファイルとしてエクスポートします。活用例：
- デバッグセッションを同僚と共有
- ワークフローをステップバイステップで文書化
- 実際のセッションからトレーニング資料を作成

### `/mcp` — 外部サービスとの連携

接続中のMCP（Model Context Protocol）サーバーの一覧表示と新規追加が可能です。CLIには**GitHub MCPサーバーが組み込み**済み — これがすべてのAPI照会を実現しています。さらに追加可能：
- **Slack** — メッセージの読み取り・検索、チャンネルへの投稿
- **データベース** — Postgres、MySQL、SQLiteを自然言語で照会
- **Jira/Linear** — プロジェクト管理ツールとの連携
- 任意のMCP互換サーバー

追加した各サーバーはエージェントに新しいツールを提供し、同じ会話内で自然に組み合わせて使用できます。

### `/init` — 新規リポジトリのブートストラップ

リポジトリに`.github/copilot-instructions.md`ファイルをスターターテンプレートとして作成します。新しいプロジェクトでCopilotを設定する際に一度実行してください — エージェントがコードベースを理解するために必要なコンテキストを提供します。

### あらゆるファイル形式 — Excel、PowerPoint、PDF...

CLIはフルシェルアクセスを持つため、**ライブラリをその場でインストール**してあらゆるファイル形式を扱えます。スプレッドシートの編集が必要？`pip install openpyxl`を実行します。スライドデッキの作成が必要？`pip install python-pptx`。事前設定は不要です。

試してみてください：
```
チームのQ1目標について3枚のシンプルなPowerPointを作成して
```
```
data.xlsxを読み込んで内容を要約して
```

## キーボードショートカット

| ショートカット | アクション |
|--------------|---------|
| `Shift+Tab` | モード切替: interactive → plan → autopilot（実験的） |
| `@` | ファイルをメンションしてコンテキストに内容を含める |
| `Ctrl+S` | 入力を保持したままコマンドを実行 |
| `Ctrl+T` | モデルの推論表示を切り替え |
| `Ctrl+U` | 現在の行の入力をすべてクリア |
| `Ctrl+W` | 前の単語を削除 |
| `Ctrl+K` | カーソルから行末まで削除 |
| `Ctrl+A` / `Ctrl+E` | 行頭 / 行末にジャンプ |
| `Ctrl+O` | 最近のタイムラインを展開（入力が空の時） |
| `Ctrl+E` | すべてのタイムラインを展開（入力が空の時） |
| `!` | Copilotをバイパスしてシェルコマンドを直接実行 |
| `↑` / `↓` | コマンド履歴をナビゲート |
| `Esc` | 現在の操作をキャンセル |
